작성자: 22B Labs · 제4의 길 (The 4th Path)

메타 설명: 지움AI의 화면, 로직, 저장, 안전장치, 배포 구조를 초보자도 이해할 수 있게 설명한 안내서입니다.

레이블/태그: 지움AI, 초보자안내, 프로젝트구조, 로컬우선, 안전설계, Next.js, 22B Labs

## 왜 이 문서가 필요한가

코드는 숲처럼 자랍니다. 처음 보는 사람은 나무 하나하나보다 길을 먼저 알아야 합니다.

지움AI는 피해자의 정보를 더 모으는 앱이 아니라, 덜 드러내고도 다음 행동을 할 수 있게 돕는 도구입니다. 그래서 이 문서는 “이 파일은 무슨 일을 하는가”보다 “사용자의 불안한 입력이 어떤 안전한 흐름을 거쳐 문서와 안내로 바뀌는가”를 중심으로 설명합니다.

앱을 처음 쓰는 피해자나 보호자라면 이 문서보다 [첫 사용자 안내서 v1.8](first-user-guide-v1.8.md)를 먼저 읽는 편이 좋습니다. 이 문서는 프로젝트 구조를 이해하려는 사람을 위한 두 번째 안내서입니다.

---

## 전체 흐름

사용자가 첫 화면에서 피해 상황을 입력하면, 지움AI는 다음 순서로 움직입니다.

1. 입력값을 읽습니다.
2. 비밀번호, 주민등록번호, 카드번호, 피해물 원본 업로드 표현이 있는지 확인합니다.
3. 차단 수준 민감정보가 있으면 결과 생성과 저장을 멈춥니다.
4. 문제가 없으면 규칙 기반으로 사건 유형을 분류합니다.
5. URL, 게시 위치, 게시자 단서, 발견 일시를 접근경로 증거목록으로 정리합니다.
6. 본인 계정, 관리자 권한, 대리 지원, 사진 속 당사자, 권한 불명확 상태를 나눠 삭제 권한을 판정합니다.
7. 삭제 요청서, 상담 준비자료, 증거 체크리스트, 공식기관 경로를 만듭니다.
8. 사용자가 원하면 현재 브라우저에만 사건을 저장합니다.
9. 저장할 때는 기본적으로 URL 경로와 민감정보를 낮춘 사본으로 바꿉니다.
10. 90일이 지난 로컬 사건은 사건 보드를 열 때 자동 정리합니다.

핵심은 단순합니다. 지움AI는 대신 삭제하지 않습니다. 사용자가 직접 움직일 수 있게 안전한 문서와 경로를 준비합니다.

## `app/` 폴더

`app/`은 사용자가 실제로 보는 페이지입니다.

- `app/page.tsx`: 첫 화면입니다. 실제 화면의 대부분은 `components/JiumApp.tsx`가 담당합니다.
- `app/dashboard/page.tsx`: 로컬 사건 보드 화면입니다. 사용자가 저장한 사건을 보고 상태를 바꿀 수 있습니다.
- `app/resources/page.tsx`: 무료 공식기관과 법률 지원 경로를 보여줍니다.
- `app/safety/page.tsx`: 디지털 성범죄, 유포 협박, 딥페이크처럼 긴급한 사건에서 먼저 볼 안전 안내입니다.
- `app/layout.tsx`: 사이트 전체의 언어, 제목, 설명 같은 공통 뼈대입니다.

초보자가 먼저 볼 파일은 `app/page.tsx`가 아니라 `components/JiumApp.tsx`입니다. `app/page.tsx`는 입구이고, 실제 흐름은 컴포넌트에 있습니다.

---

## `components/` 폴더

`components/`는 화면을 이루는 조각들입니다.

- `JiumApp.tsx`: 첫 화면, 사건 입력, 결과 생성, 로컬 저장 버튼까지 이어지는 핵심 화면입니다.
- `EvidenceLedgerInput.tsx`: 피해 게시물의 URL, 플랫폼, 게시 위치, 게시자 단서, 발견 일시, 제출 대상, 처리 상태를 여러 건으로 적는 입력 장부입니다.
- `EvidenceLedgerPanel.tsx`: 결과 화면에서 기관 제출용 접근경로 증거목록을 요약해 보여줍니다.
- `SafetyNotice.tsx`: 민감정보나 긴급 위험을 사용자에게 알려주는 안내 상자입니다.
- `RequestDraft.tsx`: 삭제 요청서나 상담 준비자료를 보여주고 복사할 수 있게 합니다.
- `ResponsePackPanel.tsx`: 안전 추적 계획, 삭제 요청 순서, 신고·고소 준비자료, 개입 선택지를 한 묶음으로 보여줍니다.
- `ResourceRouter.tsx`: 사건 유형에 맞는 공식기관을 추천합니다.
- `CaseBoard.tsx`: localStorage에 저장된 사건 목록을 보여줍니다.
- `ExternalSafeLink.tsx`: 외부 사이트로 이동하기 전에 확인창을 띄웁니다.
- `QuickExit.tsx`: 화면을 빠르게 날씨 검색 페이지로 바꾸는 버튼입니다.

여기서 가장 중요한 철학은 “버튼이 곧 권한은 아니다”입니다. 지움AI는 문서를 만들 수 있지만, 신고 제출과 삭제 요청은 사용자가 직접 확인한 뒤 공식 경로에서 해야 합니다.

특히 직접 삭제와 삭제 요청은 다르게 취급합니다. 본인 계정이나 관리자 권한이 확인된 경우에만 직접 삭제 실행 가능으로 표시하고, 사진 속 당사자일 뿐이거나 권한이 불명확하면 요청서와 공식기관 경로만 제공합니다.

## `lib/` 폴더

`lib/`는 화면 뒤에서 판단하고 문서를 만드는 곳입니다.

- `classifier.ts`: 입력 문장을 보고 사건 유형을 분류합니다. 예를 들어 딥페이크, 유포 협박, 몸캠피싱은 디지털 성범죄 긴급 경로로 보냅니다.
- `deletionAuthority.ts`: 사용자가 실제 삭제 권한을 갖고 있는지 판정합니다. "내 사진"이라는 주장만으로 직접 삭제 실행을 허용하지 않습니다.
- `evidence.ts`: 접근경로 증거목록을 정규화하고, 단일 URL 입력값을 기본 증거 항목으로 바꾸며, 기관 제출용 문서 형식으로 출력합니다.
- `pii.ts`: 주민등록번호, 비밀번호, 카드번호, 전화번호, 이메일, 피해물 원본 업로드 표현을 감지하고 가립니다.
- `requestTemplates.ts`: 플랫폼 삭제 요청서, 검색 캐시 삭제 요청서, 디지털 성범죄 상담 준비자료 같은 문서를 만듭니다.
- `responsePack.ts`: 사건 하나에 필요한 전체 대응 패키지를 만듭니다. 추적 계획, 삭제 순서, 법률상담 문서, 금지 행동이 여기에 포함됩니다.
- `publicResources.ts`: 온라인피해365센터, ECRM, KISA, 중앙디지털성범죄피해자지원센터 같은 공식기관 정보를 관리합니다.
- `caseStorage.ts`: 사건을 현재 브라우저의 localStorage에 저장하고 불러옵니다. 저장 전 URL 원문과 민감정보를 낮추고, 만료된 기록을 정리합니다.
- `export.ts`: 사건 정리 내용을 Markdown 파일로 내보낼 수 있게 만듭니다.
- `ai/`: 미래 서버형 배포에서 AI provider를 붙일 수 있는 골격입니다. GitHub Pages 공개 버전에서는 외부 AI를 호출하지 않습니다.

초보자가 수정할 때 가장 조심해야 할 파일은 `pii.ts`, `caseStorage.ts`, `responsePack.ts`입니다. 이 세 파일은 사용자 안전과 직접 연결됩니다.

## 사건 분류

사건 분류는 “정답 맞히기 게임”이 아닙니다. 안전한 첫 경로를 고르는 일입니다.

예를 들어 `협박`이라는 단어가 들어오면 위험도를 높게 봅니다. 딥페이크, 유포, 성적 이미지, 몸캠피싱, 그루밍 같은 단어가 함께 있으면 앱 안에서 해결하려 하지 않고 전문기관 연결을 우선합니다.

분류 결과에는 다음 정보가 포함됩니다.

- 사건 유형
- 위험도
- 삭제 가능성
- 추천 경로
- 즉시 할 일
- 증거 체크리스트
- 7일/30일/90일 재확인 일정
- 법률 대리나 삭제 보장이 아니라는 안내

이 구조는 피해자가 처음부터 법률 용어를 알 필요 없게 만들기 위한 완충 장치입니다.

## 민감정보 차단

지움AI는 다음 정보가 들어오면 조심스럽게 멈춥니다.

- 비밀번호
- 주민등록번호
- 카드번호
- 피해 이미지나 영상 원본 업로드 표현

전화번호와 이메일은 완전히 막지는 않지만 마스킹 미리보기로 가립니다. 피해자가 삭제 요청서에 쓸 정보와 앱이 저장해도 되는 정보는 다르기 때문입니다.

중요한 원칙은 하나입니다. 피해자를 돕는 도구가 피해자의 비밀을 더 오래 붙잡고 있으면 안 됩니다.

## 로컬 저장

사건 보드는 서버 DB가 아니라 현재 브라우저의 localStorage를 사용합니다.

이 말은 두 가지 뜻입니다.

- 장점: 서버로 사건이 올라가지 않습니다.
- 주의점: 같은 브라우저를 쓰는 사람이 기록을 볼 수 있습니다.

그래서 현재 저장 정책은 한 번 더 낮춰져 있습니다.

- URL은 원문 경로를 저장하지 않고 도메인 중심으로 줄입니다.
- 개인 기기에서 기관 제출용으로 정확한 URL 보관이 꼭 필요할 때만 사용자가 명시적으로 선택할 수 있습니다.
- 접근경로 증거목록 안의 URL도 같은 정책을 따릅니다.
- 전화번호, 이메일, 주민등록번호, 비밀번호 같은 값은 마스킹합니다.
- 저장용 요청서와 대응 패키지는 낮춰진 입력값으로 다시 만듭니다.
- 90일이 지난 사건은 사건 보드를 열 때 자동 정리합니다.
- 사건 보드에서 개별 삭제와 전체 삭제를 할 수 있습니다.

공용 PC라면 저장하지 않는 것이 가장 안전합니다.

## 대응 패키지

대응 패키지는 사용자가 오늘 해야 할 일을 한 묶음으로 정리한 결과입니다.

포함되는 내용은 다음과 같습니다.

- 접근경로 증거목록
- 안전 추적 계획
- 삭제 요청 순서
- 피해자 직접 삭제 희망 플랜
- 삭제 권한 판정과 차단되는 행동
- 안전한 개입 선택지
- 유출자 특정 단서 정리
- 범죄유형별 피해 확산 방지 매트릭스
- 경찰 신고 준비서
- 형사 고소 상담 준비자료
- 무료 법률상담 메모
- 공식기관과 법률 지원 서비스 연계

특히 “안전한 개입 선택지”는 세 가지로 나뉩니다.

- 안전한 공식 조치: 공식기관 상담, 플랫폼 신고, 검색 제외 요청
- 상담 후 결정: 고소, 손해배상, 변호사 상담처럼 법률 검토가 필요한 선택
- 하지 말아야 할 일: 피해물 다운로드, 초대코드 공유, 자동 대량 신고, 신상털이, 가해자와 직접 협상

이 구분은 제품의 심장에 가깝습니다. 빠른 행동보다 안전한 행동이 먼저입니다.

## 공식기관 연결

공식기관 정보는 `lib/publicResources.ts`에 있습니다.

현재 연결되는 주요 경로는 다음과 같습니다.

- 중앙디지털성범죄피해자지원센터
- 여성긴급전화 1366
- 온라인피해365센터
- KISA 개인정보침해 신고센터
- 경찰청 사이버범죄 신고시스템 ECRM
- 방송미디어통신심의위원회
- 개인정보 포털 지우개 서비스
- 털린 내 정보 찾기
- 대한법률구조공단
- 대한변협 나의 변호사
- 로톡, 로앤굿

민간 법률 플랫폼은 선택지일 뿐입니다. 지움AI는 공식·무료 경로를 먼저 보여주는 쪽으로 설계되어 있습니다.

## 테스트

테스트는 “이 앱이 똑똑한가”보다 “위험한 실수를 하지 않는가”를 확인합니다.

- `classifier.test.ts`: 사건 유형이 안전하게 분류되는지 확인합니다.
- `pii.test.ts`: 비밀번호, 주민등록번호, 카드번호 같은 정보가 감지되는지 확인합니다.
- `responsePack.test.ts`: 신고서, 법률상담 자료, 금지 행동 안내가 생성되는지 확인합니다.
- `deletionAuthority.test.ts`: 직접 삭제 실행 권한이 본인 계정·관리자 권한일 때만 열리고, 사진 속 당사자 또는 권한 불명확 상태에서는 차단되는지 확인합니다.
- `evidence.test.ts`: 접근경로 증거목록이 기관 제출용 문서로 정리되는지 확인합니다.
- `caseStorage.test.ts`: 로컬 저장 전에 URL과 민감정보가 낮춰지고, 만료된 사건이 정리되는지 확인합니다.
- `publicResources.test.ts`: 공식기관 라우팅이 작동하는지 확인합니다.
- `requestTemplates.test.ts`: 요청서 템플릿이 필요한 내용을 포함하는지 확인합니다.
- `aiRouter.test.ts`: 기본값에서 외부 AI가 아니라 rule-based fallback을 쓰는지 확인합니다.

검증 명령은 세 가지입니다.

```bash
npm run typecheck
npm test
npm run build
```

GitHub Pages 배포 전에는 다음 조건도 확인합니다.

```powershell
$env:GITHUB_PAGES='true'; $env:NEXT_PUBLIC_BASE_PATH='/jium-ai'; npm run build
```

## 배포

배포는 GitHub Pages workflow가 담당합니다.

- workflow 파일: `.github/workflows/deploy-pages.yml`
- main 브랜치에 push하면 자동 실행됩니다.
- `GITHUB_PAGES=true`일 때 Next.js가 정적 export 모드로 빌드됩니다.
- 결과물은 `out/` 폴더 기준으로 GitHub Pages에 올라갑니다.

공개 주소는 다음과 같습니다.

```txt
https://sinmb79.github.io/jium-ai/
```
