# 지움AI v1.4 전체 재검토 보강 기록

작성자: 22B Labs · 제4의 길 (The 4th Path)

메타 설명: 지움AI 최종 재검토에서 확인한 보안·사용성 누락과 보완 내역입니다.

레이블/태그: 지움AI, 최종검토, 보안강화, 민감정보차단, 외부연계, 로컬우선, 피해자지원, 22B Labs

---

## 왜 다시 봤는가

지움AI의 위험은 기능 부족보다 방향의 과잉에서 생깁니다.

피해자를 돕겠다는 프로그램이 피해 URL을 자동으로 접속하거나, 피해물 원본을 다루거나, 유출자를 임의로 특정하려 들면 도움의 모양을 한 2차 피해가 됩니다. 그래서 이번 재검토는 “더 많이 자동화할 수 있는가”보다 “덜 드러나고도 움직일 수 있는가”를 기준으로 삼았습니다.

---

## 발견한 누락

### 1. 차단 수준 민감정보가 결과 화면에 다시 나타날 수 있음

기존 구현은 비밀번호, 주민등록번호, 카드번호, 피해물 원본 업로드 표현을 감지하고 로컬 저장 버튼을 막았습니다. 그러나 사용자가 결과 만들기를 누르면 요청서와 신고 준비자료는 화면에 생성될 수 있었습니다.

피해자 지원 도구에서 원문이 서버로 가지 않는 것만으로는 충분하지 않습니다. 화면에 다시 펼쳐지는 것도 위험입니다.

보완:

- 차단 수준 민감정보가 있으면 결과 생성 자체를 중단
- 요청서, 신고서, 형사 고소 상담자료, 로컬 저장 모두 생성 전 차단
- 차단 사유를 폼 안에서 즉시 안내
- 외부 AI adapter도 차단 상태에서는 rule-based fallback만 사용

### 2. 외부 사이트 이동 확인 흐름이 일부 페이지에만 적용됨

결과 화면의 기관 연계는 확인창을 거쳤지만, 공식기관 목록과 긴급 안전 페이지의 외부 링크는 일반 새 창 링크였습니다.

보완:

- `ExternalSafeLink` 공통 컴포넌트 추가
- 공식기관 목록, 긴급 안전 페이지, 결과 화면의 외부 이동을 동일한 확인 흐름으로 통일
- 사건 내용과 피해 URL이 자동 전달되지 않는다는 문구를 이동 전 표시

### 3. 사건 보드 삭제가 너무 즉시적임

로컬 저장은 안전하지만, 피해자가 힘들게 정리한 사건 기록을 실수로 지우는 것도 실제 피해입니다.

보완:

- 로컬 사건 삭제 전 확인창 추가
- 사건 카드 설명에 신고·고소 준비자료와 피해 확산 방지 매트릭스 포함 사실 표시

### 4. 개발 문서의 현재 상태가 오래됨

`DEVELOPMENT.md`에는 아직 앱 코드와 Git 저장소가 없다고 적혀 있었습니다.

보완:

- 현재 구현 상태, GitHub repo, 공개 페이지, v1.2~v1.4 문서 목록을 반영

## 남겨둔 경계

지움AI는 다음을 하지 않습니다.

- 피해 URL 자동 접속
- 플랫폼 로그인 자동화
- 자동 삭제 요청 제출
- 자동 대량 신고
- 피해물 원본 업로드 또는 분석
- 유출자 신원 단정
- 법률 대리 또는 삭제 성공 보장

이 경계는 기능 부족이 아니라 제품의 윤리적 외벽입니다.
