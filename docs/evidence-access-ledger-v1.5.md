# 지움AI 접근경로 증거목록 설계 v1.5

작성자: 22B Labs · 제4의 길 (The 4th Path)

메타 설명: 피해 게시물 접근경로를 안전하게 정리해 공식기관 제출자료로 바꾸는 지움AI 보강 설계.

레이블/태그: 지움AI, 접근경로, 증거목록, 디지털성범죄, 삭제지원, 로컬우선, 피해자지원, 22B Labs

---

## 왜 이 기능이 필요한가

삭제지원과 수사는 감정의 호소만으로 움직이지 않습니다. 기관과 플랫폼은 확인 가능한 경로를 필요로 합니다.

피해자의 유출 사진이나 영상이 어디에 올라왔는지, 어떤 URL인지, 게시자 단서가 무엇인지, 언제 발견했는지, 어디에 제출할 것인지가 정리되어야 실제 구제의 문턱을 넘을 수 있습니다.

다만 이 지점이 가장 위험합니다. 프로그램이 피해물을 자동으로 열람하거나 크롤링하거나 다운로드하면, 도움의 이름으로 2차 피해를 만들 수 있습니다. 그래서 지움AI의 보강 방향은 자동 수집이 아니라 안전한 접근경로 장부입니다.

## 반영한 제품 원칙

1. 피해물 원본은 저장하지 않습니다.
2. URL, 게시 위치, 게시자 ID, 발견 일시, 제출 대상, 처리 상태만 기록합니다.
3. 여러 접근경로를 한 사건 안에 누적할 수 있습니다.
4. 기관 제출용 문서, 경찰 신고 준비서, 형사 고소 상담자료, 내보내기 파일에 같은 증거목록을 반영합니다.
5. 로컬 보드는 기본적으로 URL 경로를 숨깁니다.
6. 개인 기기에서 기관 제출을 위해 필요할 때만 정확한 URL 보관을 사용자가 명시적으로 켤 수 있습니다.

## 구현 위치

- `lib/types.ts`: `EvidenceItem`, `EvidenceStatus`, `CaseInput.evidenceItems`, `keepExactUrlsForSubmission`
- `lib/evidence.ts`: 접근경로 정규화, 기본 URL에서 증거목록 생성, 문서 출력 포맷
- `components/EvidenceLedgerInput.tsx`: 여러 접근경로를 입력하는 폼
- `components/EvidenceLedgerPanel.tsx`: 결과 화면의 기관 제출용 증거목록 요약
- `lib/requestTemplates.ts`: 삭제 요청서와 디지털성범죄 상담 준비자료에 증거목록 포함
- `lib/responsePack.ts`: 경찰 신고 준비서, 형사 고소 상담자료, 무료 법률상담 메모에 증거목록 포함
- `lib/export.ts`: Markdown 내보내기 문서에 증거목록 포함
- `lib/caseStorage.ts`: 기본 저장 시 URL 경로 숨김, opt-in 시 정확한 URL 보관

## 남겨둔 경계

지움AI는 여전히 다음을 하지 않습니다.

- 피해 URL 자동 접속
- 피해물 원본 다운로드
- 로그인 자동화
- 무단 크롤링
- 자동 기관 제출
- 유출자 신원 단정

이 경계는 약함이 아니라 피해자 지원 도구가 지켜야 할 윤리적 외피입니다.

## 검증

- `npm run typecheck`
- `npm test`
- `npm run build`

접근경로 증거목록은 문서에 정확히 들어가야 하지만, 로컬 보드에는 사용자가 선택하지 않는 한 정확한 경로가 남지 않아야 합니다. 이 두 조건을 테스트로 고정했습니다.

## GitHub 반영

이 보강은 `sinmb79/jium-ai` 레포의 README, 개발 문서, 초보자 안내서, 피해자 직접 삭제 희망 플로우와 함께 관리합니다. 공개 페이지에서는 사용자가 별도 서버나 API 키 없이 접근경로 증거목록을 만들고, 필요한 문서를 내려받거나 복사해 공식기관 절차로 직접 옮길 수 있어야 합니다.
