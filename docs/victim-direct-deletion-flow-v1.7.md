# 피해자 직접 삭제 희망 플로우 v1.7

작성자: 22B Labs · 제4의 길 (The 4th Path)

메타 설명: 피해자가 직접 삭제를 원할 때 안전하게 움직이도록 돕는 지움AI 실행 플로우입니다.

태그: 지움AI, 피해자삭제, 개인정보, 디지털성범죄, 증거보존, 로컬우선

---

## 왜 이 기능이 필요한가

피해자는 보통 "지워주세요"라고 말하지만, 실제로 필요한 것은 하나의 삭제 버튼이 아닙니다.

삭제 전 증거가 사라지면 이후 거절, 재유포, 수사, 심의 절차가 어려워집니다. 반대로 증거를 모으겠다고 피해물을 다시 내려받거나 주변에 보내면 2차 피해가 커집니다. 지움AI의 역할은 그 사이를 좁히는 것입니다.

> 혼자 지우게 하지 않는다. 그러나 피해자의 손에서 통제권을 빼앗지도 않는다.

---

## 구현 범위

v1.7은 `ResponsePack.victimDeletionPlan`을 추가해 사건별 직접 삭제 실행 플랜을 생성합니다.

- 일반 개인정보·사진·이미지 노출: 직접 삭제 요청 가능
- 자기 게시물 삭제: 직접 삭제, 지우개 서비스, 검색 캐시 제거 순서 제공
- 검색 노출 제거: 원본 조치와 검색엔진 조치 분리
- 디지털성범죄·유포협박: 피해자 단독 처리보다 전문기관 삭제지원 우선

앱은 삭제를 자동 수행하지 않습니다. 자동 제출, 자동 접속, 자동 크롤링, 로그인 자동화도 하지 않습니다.

---

## 화면 구조

`components/ResponsePackPanel.tsx`에 "피해자 직접 삭제 희망 플랜" 패널을 추가했습니다.

패널은 다음 정보를 보여줍니다.

- 직접 요청 가능 여부
- 첫 원칙
- 사건별 실행 단계
- 공식기관으로 격상해야 하는 신호
- 기록 원칙
- 자동화 경계
- 직접 삭제 요청 문안 복사

---

## 데이터 구조

`lib/types.ts`에 다음 타입을 추가했습니다.

- `VictimDeletionStep`
- `VictimDeletionPlan`

`ResponsePack`에는 `victimDeletionPlan` 필드가 포함됩니다.

각 단계는 주체, 시점, 사용자 행동, 필요 자료, 완료 신호, 다음 사건 상태를 갖습니다. 삭제 완료와 사용자 직접 확인 완료를 분리하기 위해 `nextStatus`를 명시합니다.

---

## 안전 원칙

1. 삭제 전 URL, 게시 위치, 발견 일시를 기록합니다.
2. 피해물 원본은 지움AI에 올리지 않습니다.
3. 디지털성범죄 사건은 전문기관 삭제지원이 먼저입니다.
4. 플랫폼 요청, 검색엔진 요청, 공적기관 신고를 한 흐름으로 섞지 않습니다.
5. 삭제 완료처럼 보여도 7일, 30일, 90일 재확인을 남깁니다.
6. 재노출은 기존 성공을 덮어쓰지 않고 새 접근경로로 추가합니다.

---

## 검증 기준

- 무단 사진·이미지 설명이 개인정보 노출로 분류됩니다.
- 디지털성범죄 사건은 직접 요청보다 전문기관 우선 플랜을 생성합니다.
- 일반 사진·개인정보 노출은 직접 삭제 요청 플랜을 생성합니다.
- 복사 문안에는 접근경로 증거목록이 포함됩니다.
- 내보내기 문서에도 직접 삭제 플랜이 포함됩니다.

작은 기능처럼 보이지만, 피해자에게는 여기서 세계가 갈립니다. 삭제는 끝이 아니라 통제권 회복의 시작입니다.
